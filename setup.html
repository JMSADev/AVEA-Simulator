Ativar sincronização entre os aparelhos

Sem esse passo, cada jogador só vê os dados salvos no próprio navegador. Com ele, todo mundo — cada um no seu computador ou celular — enxerga as mesmas fichas e a mesma cena de combate. A sincronização usa o Firebase Realtime Database (gratuito no plano usado aqui). As chaves do projeto já vêm prontas dentro de js/config.js, então só falta um passo: liberar a leitura/escrita no banco. Só uma pessoa do grupo (o mestre, de preferência) precisa fazer isso.

Passo 1 — Abrir as regras do Realtime Database

  1. Entre no console do Firebase com a conta Google usada pra criar o projeto.
  2. Abra o projeto do grupo (o mesmo cujo databaseURL aparece em js/config.js).
  3. No menu à esquerda, vá em Build → Realtime Database.
  4. Clique na aba Rules (Regras).

Passo 2 — Liberar leitura e escrita

Apague o conteúdo e cole isto no lugar:

{
  "rules": {
    ".read": true,
    ".write": true
  }
}

Clique em Publish (Publicar). Pronto — o site já consegue ler e gravar no banco.

Passo 3 — Testar

Abra ficha.html em dois navegadores (ou computador + celular), logue com o mesmo personagem em um deles, edite algo e recarregue o outro — a mudança deve aparecer. Se não aparecer, abra o Console do navegador (F12) e veja se há algum erro relacionado a firebase ou permission_denied: isso normalmente indica que o Passo 2 não foi publicado corretamente.

Sobre segurança

Com ".read": true, ".write": true, o banco fica público e sem autenticação: qualquer pessoa que souber a URL do projeto (que aparece em js/config.js, exposto no navegador de qualquer jogador) consegue ler ou sobrescrever os dados diretamente, sem passar pelo site. As senhas dos personagens também trafegam em texto puro. Isso é adequado pra travar bisbilhotice casual entre amigos — não use isso pra guardar nada sensível de verdade.
