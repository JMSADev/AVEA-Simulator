Ativar sincronização entre os aparelhos

Sem esse passo, cada jogador só vê os dados salvos no próprio navegador. Com ele, todo mundo — cada um no seu computador ou celular — enxerga as mesmas fichas e a mesma cena de combate. Usa um serviço gratuito (jsonblob.com), sem precisar criar conta. Leva uns 2 minutos. Só uma pessoa do grupo precisa fazer isso — depois, todo mundo usa o mesmo arquivo de configuração.

Passo 1 — Criar o "blob" de personagens

  1. Abra jsonblob.com em uma nova aba.
  2. Apague o conteúdo de exemplo do editor e cole isto no lugar: 
    {"mestreSenha":"","fichas":[],"inimigos":[]}
  3. Clique em "Save new JSONBlob" (ou botão equivalente de salvar).
  4. A página vai mostrar uma URL parecida com https://jsonblob.com/1234abcd-.... Copie só a parte final (o ID depois da última barra).

Passo 2 — Criar o "blob" de combate

Repita o processo acima, mas cole este conteúdo:

{"combatentes":[],"turnoIdx":-1,"log":[]}

Salve e copie o segundo ID.

Passo 3 — Colar os IDs no arquivo js/config.js

Abra o arquivo js/config.js (dentro da pasta do site) em qualquer editor de texto e troque estas duas linhas, colando seus IDs no lugar de COLOQUE_SEU_ID_AQUI:

window.ARTON_BLOB_PERSONAGENS = 'COLOQUE_SEU_ID_AQUI'; // ID do passo 1
window.ARTON_BLOB_COMBATE     = 'COLOQUE_SEU_ID_AQUI'; // ID do passo 2

Salve o arquivo, suba a pasta inteira (com essa alteração) pro lugar onde todo mundo acessa — por exemplo, uma página no GitHub Pages — e pronto: qualquer pessoa que abrir o site vai estar lendo e escrevendo nesses mesmos dois arquivos na nuvem.

Sobre segurança

Esse armazenamento é público e sem autenticação: qualquer pessoa que souber os dois IDs consegue ler ou sobrescrever os dados diretamente, sem passar pelo site. As senhas dos personagens também trafegam em texto puro. Isso é adequado pra travar bisbilhotice casual entre amigos — não use isso pra guardar nada sensível de verdade.
